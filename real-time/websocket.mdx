## Endpoint

```
wss://polychadsbot.xyz/ws/alerts?api_key=YOUR_KEY
```

## Message Types

### `connected`

Sent once when the connection is established.

```json
{
  "type": "connected",
  "message": "Connected to Polychads alert stream",
  "tier": "pro"
}
```

### `alert`

Sent whenever a new insider alert is detected. The `data` field contains the full [Alert object](/reference/alert-object).

```json
{
  "type": "alert",
  "data": {
    "id": 2221,
    "alert_type": "fresh",
    "side": "BUY",
    "outcome": "Yes",
    "entry_price": 0.42,
    "usdc_value": 2100.0,
    "signal_score": 72,
    "event_title": "Will ETH hit $5000 by April?",
    "category": "crypto",
    "created_at": "2026-02-27 12:00:15"
  }
}
```

### `pong`

Response to your `ping` heartbeat messages.

```json
{
  "type": "pong"
}
```

## Heartbeat

You **must** send a `ping` message at least every 30 seconds to keep the connection alive. The server responds with a `pong`.

```javascript
setInterval(() => {
  if (ws.readyState === WebSocket.OPEN) {
    ws.send("ping");
  }
}, 25000);
```

## Reconnection

If the connection drops, implement exponential backoff:

```javascript
let reconnectDelay = 1000;

ws.onclose = () => {
  setTimeout(() => {
    reconnectDelay = Math.min(reconnectDelay * 2, 30000);
    connect(); // your connect function
  }, reconnectDelay);
};

ws.onopen = () => {
  reconnectDelay = 1000; // reset on success
};
```

<Warning>
The WebSocket endpoint uses `api_key` as a query parameter (not the `Authorization` header). Make sure to URL-encode your key if it contains special characters.
</Warning>
